◆ bch_code_decode()

void bch_code_decode ( uint8_t * m,
uint8_t * em )

Decodes a supposed BCH code word.

The decoding procedure is described in BCH code.

This function takes as input a pointer to a vector that is the received BCH code word and return a pointer to vector that is the original message being encoded (the information bits). The decoding procedure follows the steps described by Shu. Lin and Daniel J. Costello in Error Control Coding: Fundamentals and Applications [lin1983error]. In the following the vectors are considered as polynomials.

  • The decoding procedure consists of four major steps:
    1. The first step is the computation of 2 * PARAM_DELTA (with PARAM_DELTA being the error correcting capacity of the BCH code) syndromes.
    2. The second step is the computation of the error-location polynomial \( \sigma\)(x) from (defined in BCH code) the syndromes computed in the first step.
    3. The third step is to find the error-location numbers (defined in BCH code) by calculating the roots of the polynomial \( \sigma\)(x) and returning their inverses.
    4. The fourth step is the correction of errors in the received polynomial
Parameters
[out]mPointer to an array that is the decoded code word
[in]emPointer to an array that is supposed to be a code word

◆ bch_code_encode()

void bch_code_encode ( uint8_t * em,
uint8_t * m )

Encodes a message using the BCH code.

This function encode a message of PARAM_K bits to a code word of size PARAM_N1 bits. Following [lin1983error] (Chapter 4 - Cyclic Codes), we perform a systematic encoding using a linear (PARAM_N1 - PARAM_K)-stage shift register with feedback connections based on the generator polynomial of the BCH code.

For reasons of clarity and comprehensibility, we achieve the encoding process in four steps (the vectors are considered as polynomials):

  1. First, we convert the input m which contains the message to be encoded to an array where each coordinate of the input is stored in an unsigned char.
  2. Second, we parse the precomputed generator polynomial \( g \)(x) of the BCH code (defined in BCH code) from its hexadecimal representation.
  3. Third, we use an LFSR as described in [lin1983error] to encode the message and build the code word.
  4. Fourth, we convert the code word to an array of unsigned chars.
Parameters
[out]emPointer to an array that is the encoded message
[in]mPointer to an array that is the message

◆ gf_generation()

void gf_generation ( gf_tables * gf_tables)

Generates the Log and Anti-Log tables for a \( GF(2^{10}) \).

We will be working in the Galois Field \( GF(2^{m}) \) (with \( m = 10 \)), this field can be constructed using the primitive polynomial \( p(X) = 1 + X^3 + X^{10}\) of degree \( 10 \) over \( GF(2) \). Let \(p(\alpha) = 1 + \alpha^3 + \alpha^{10} = 0\), then using the identity \(\alpha^{10} = \alpha^3 + 1\), we can construct a polynomial representation of the elements of \( GF(2^{10}) \). We can construct a binary representation using the polynomial representation. In fact, let \( a_0 + a_1 \alpha +a_2 \alpha^2 + ... + a_9 \alpha^{9}\) be the polynomial representation of \( \beta\), then \( \beta \) can be represented by the binary 10-tuple \( (a_0, a_1, ... , a_9) \). In the other hand, we have a power representation, in fact \( GF(2^{10}) = \{0, 1, \alpha, \alpha^2, ..., \alpha^{2^{10} - 2} \}\). Thus, we have three representations of \( GF(2^{10}) \); a polynomial representation, a binary representation and a power representation. To perform computations (additions and multiplications) in a Galois Field, we use look-up tables. As in [morelos2006art], we will use Log and Anti-Log tables:

  1. Anti-Log table \( a(i)\): is used when performing additions. At index \( i \), it gives the value of the binary vector that corresponds to the element \( \alpha^i \).
  2. Log table \( log(i) \): is used when performing multiplications. At index \( i \), it gives \( log(i) \) which is the power of \( \alpha^{log(i)} \) that corresponds to the binary vector represented by the integer \( i \).

Such that the following relation holds: \( \alpha^{log(i)} = a(i)\)

Parameters
[in]gf_tablesPointer to a Log and Anti-Log tables
